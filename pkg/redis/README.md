# Пакет для работы с Redis

Подключение производится через автомонтирование:

```go
type Cron struct {
	redis  redis.IRedis
}

func NewCron(redis redis.IRedis) *Cron {
	return &Cron{redis: redis}
}
```

Пакет содержит следующие методы для работы с кешом:
- `Set` - Запись в кеш произвольных данных
- `GetInt` - Получение из кеша значение с типом int
- `GetFloat` - Получение из кеша значение с типом int
- `GetString` - Получение из кеша значение с типом float64
- `GetStruct` - Получение данных из кеша и копирование их в структуру
- `SetAndGet` - Запись в кеш данных если их там нет, возвращение кешированных данных
- `Lock` - Блокировка по средствам редиса
- `Unlock` - Снятие блокировки 

Пакет работает с префиксами, префикс задается через конфиг yaml. 
Префиксы нужны, что бы несколько приложений (например dev, dev1, dev2 и т.д.) работали с одним редисом и не мешали друг другу.
Такой подход упрощает развертывание копий приложения на несколько стендов.

### Запись в кеш
```go
key := "someKeyName"
value := 123 // - может быть любым типом: числом, строкой, структурой и т.д.

err := redis.Set(ctx, key, value)
```

### Чтение кеша
Допустим мы знаем что записиали в кеш под ключем `someKeyName` числовое значение. 
Это значение можно получить сразу числом, что бы не заниматься преобразованием типов, 
для этого нужно использовать специально созданный для этого метод `GetInt()`.
В случае ошибка разработчика, например если хранимое в кеше значение не числовое, метод отдаст ошибку.

```go
key := "someKeyName"
value, exists, err := redis.Get(ctx, key)
if err != nil {
	return common.Wrap(err, "error getting the int type")
}
if !exists() {
    return common.Wrap(err, "empty")
}

fmt.Println(value) // Выведет: 123
```

### SetAndGet
Допустим есть значение которое нужно закешировать и всегда отдавать из кеша. 
Можно написать метод, который будет проверять наличие значение в кеше: если кеш есть - то отдавать значение, если кеша нет - то кешировать это значение и отдавать уже закешированное.
По такому принципе работает метод `GetAndSet`.

```go
err := redis.SetAndGet(context.Background(), &res, func() (int, error) {
	// Тут должна быть логика для расчета значения который мы положим в кеш
    return 1, nil // Допустим, в результате операций выше посчиталось значение 1, которое пакет положит в кеш.
}, key, time.Hour)
```
По сути дела, метод вытаскивает числовое значение из кеша если оно там есть, если значения нет, то выполняет код и результат кладет в кеш.

### Блокировка на уровне redis
Допустим некий код необходимо выполнить только один раз, но мы знаем что 
наше приложение развернуто в кубернетесе и имеет несколько подов, 
в таком случае запущенный код будет выполняться на всех подах (экземплярах) приложения.
Решением этой задачи будет выполнить задачу на одном экземпляре, а на всех других экземплярах - заблокировать выполнение.

```go
ok, err := c.redis.Lock(ctx, redisLockKey, 3*time.Second)
if err != nil {
    return common.Wrap(err, "lock err")
}
if !ok {
	return errors.New("already started") // ошибка возникнет только в том случае если код уже начал выполняться на другом поде
}

// Код который необходимо выполнить только на одном поде
```
Досрочно снять блокировку:
```go
err := r.redis.Unlock(ctx, key)
```
