# Планировщик задач

Инициализация планировщика происходит через автомонтирование в
internal/app/app.go:54. Добавление новых задач через метод `OnStart:` в файле internal/app/worker.go:54.

Пример:

```go
OnStart: func(ctx context.Context) error {
    go func() {
        c.AddJob(ctx, "*/1 * * * *", testJobRun)
        c.Start()
    }()
    return nil
}
```

`c.AddJob()` - отвечает за постановку задачи, аругемнтов у методов всего три:
- `ctx` - контекст
- `*/1 * * * *` - расписание запуска задачи 
  - минуты; часы; день; месяц; день недели
- `testJobRun` - метод для запуска задачи

Пример метода для запуска задачи: 

```go
testJobRun := func Run(ctx context.Context) error {
	fmt.Println("Task is run") // бизнес-логика тут
	return nil
}
```

Примеры расписания запуска задчи:
- `*/1 * * * *` - каждую 1 минут
- `*/1 * * * *` - в 01 минут каждого часа (00:01, 01:01, 02:01 и т.д.)
- `* */1 * * *` - каждый час
- `30 */2 * * *` - в 30 минут каждые 2 часа (00:30, 02:30, 04:30 и т.д.)
- `0 0 1 * *` - 1 числа каждого месяца в 00:00
- `0 14 15 3 *` - 15 марта в 14:00
- `0 0 * * 5` - в 00:00 каждую пятницу (1 понедельник, 2 вторник, 3 среда, 4 четверг, 5 пятница, 6 суббота, 0 или 7 воскресенье)

В реальности, метод для запуска выносится в домен бизнес логики.

Пакет `scheduler` должен запускаться и работать кубернетесом в одном экземпляре. 
В случае встраивания и запуска пакета внутри основного приложения он всё равно запуститься в одном экземпляре для предотвращения дубликатов старта.
Дубликаты старта предотвращаются с помощью блокировки на redis.
